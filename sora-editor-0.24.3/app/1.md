当前项目的语法高亮是“`Language`(语言实现) → `AnalyzeManager`(分析器) → `Styles/Spans`(样式结果) → `CodeEditor`/`EditorRenderer`(渲染)”这条链路，并且内置了多套高亮引擎：TextMate、Monarch、Tree-sitter，以及一个示例/基础的 Java 语言高亮。

**整体高亮主链路（所有语言实现共用）**
- 入口：`CodeEditor#setEditorLanguage()` 会把 `Language#getAnalyzeManager()` 的接收器设置为 `EditorStyleDelegate`，并触发首次 `reset()` 分析（`editor/src/main/java/.../CodeEditor.java:952-1004`）。
- 分析器输出：分析器通过 `StyleReceiver#setStyles()` / `updateStyles()` 把 `Styles` 推给编辑器（接口定义在 `editor/src/main/java/.../lang/analysis/StyleReceiver.java:44-76`）。
- 落地到编辑器：`EditorStyleDelegate` 把 `Styles` 切回 UI 线程并调用 `CodeEditor#setStyles()` 或 `CodeEditor#updateStyles()`（`editor/.../widget/EditorStyleDelegate.java:96-137`，`editor/.../widget/CodeEditor.java:4215-4237`）。
- 渲染使用：`EditorRenderer` 在绘制每一行时从 `Styles.spans` 拿 `Spans.Reader`，并把 `spans.getSpansOnLine(line)` 交给 `TextRow` 绘制（`editor/.../widget/EditorRenderer.java:381-437`）。

**核心数据结构（高亮最终长什么样）**
- `Styles`：承载整份分析结果（`spans` + `blocks` 折叠块 + 行级装饰等），并提供插入/删除时的跨度调整（`editor/.../lang/styling/Styles.java:51-279`）。
- `Spans`：按“行→Span列表”提供读取接口（`read()`/`getSpansOnLine()`），渲染侧只认这个抽象（`editor/.../lang/styling/Spans.java:34-131`）。
- `Span`/`TextStyle`：每个 span 记录起始列和样式（颜色类型、粗斜体等）；渲染时由当前 `EditorColorScheme` 把“颜色类型”映射成实际 ARGB 颜色。

**本项目包含的几套语法高亮实现**

1) **TextMate（tm4e + Oniguruma）**
- 语言类：`TextMateLanguage`（`language-textmate/.../TextMateLanguage.java`），内部创建 `TextMateAnalyzer`（`TextMateLanguage.java:198-219`）。
- 分析器：`TextMateAnalyzer extends AsyncIncrementalAnalyzeManager`，核心是 `grammar.tokenizeLine2(...)` 得到 token，再转成 `Span`（`language-textmate/.../TextMateAnalyzer.java:68-257`）。
- 主题/颜色映射关键点：
  - analyzer 用 `foreground + 255` 作为“动态颜色类型”，写进 `TextStyle`（`TextMateAnalyzer.java:243-246`）。
  - `TextMateColorScheme#getColor(type>=255)` 再用 `theme.getColor(type-255)` 取主题色并缓存（`language-textmate/.../TextMateColorScheme.java:315-338`）。
- 语法/配置来源（demo 里从 assets 加载）：
  - 主题：`app/src/main/assets/textmate/*.json`，加载逻辑在 `MainActivity.loadDefaultTextMateThemes()`（`app/.../MainActivity.kt:344-363`）。
  - 语法：`app/src/main/assets/textmate/languages.json` + 各语言 `syntax/*.tmLanguage*`，加载逻辑在 `MainActivity.loadDefaultTextMateLanguages()`（`MainActivity.kt:370-372`）。
  - `language-configuration.json`（括号、折叠 markers 等）会在 `GrammarRegistry.doLoadGrammar()` 里读入并绑定到 scope（`language-textmate/.../registry/GrammarRegistry.java:202-220`）。

2) **Monarch（VS Code Monarch tokenizer 思路）**
- 分析器：`MonarchAnalyzer extends AsyncIncrementalAnalyzeManager`（`language-monarch/.../MonarchAnalyzer.kt:59-205`），对每行调用 monarch tokenizer 得到 token，再转 `Span`。
- 主题/颜色映射同样走 “type>=255 动态色”：
  - `MonarchColorScheme#getColor(type>=255)` 从 `ThemeModel.value.colorMap.getColor(type-255)` 取色并缓存（`language-monarch/.../MonarchColorScheme.kt:204-225`）。
- 配置来源（demo 里从 assets 加载 textmate 同款主题/语言配置）：
  - `MainActivity.loadDefaultMonarchThemes()` 直接复用 `app/src/main/assets/textmate/*.json`（`MainActivity.kt:393-410`）。
  - `MainActivity.loadDefaultMonarchLanguages()` 里用 DSL 注册各语言，并复用 `textmate/*/language-configuration.json`（`MainActivity.kt:415-439`）。

3) **Tree-sitter（解析树 + query 高亮）**
- 语言类：`TsLanguage`（`language-treesitter/.../TsLanguage.kt:50-114`），内部 `TsAnalyzeManager` 负责解析/更新。
- 分析器：`TsAnalyzeManager` 维护 tree-sitter `TSParser/TSTree`，在初始化/增量编辑后重建 `LineSpansGenerator`（`language-treesitter/.../TsAnalyzeManager.kt:45-199`）。
- 关键：`LineSpansGenerator` 在 `Spans.Reader.moveToLine()` 里按行范围跑 `TSQueryCursor`，把 captures 转 span，并做 LRU 缓存（`language-treesitter/.../LineSpansGenerator.kt:45-250`）。
- 高亮规则来源：`.scm` query 文件（demo 里在 assets）：
  - `app/src/main/assets/tree-sitter-queries/java/highlights.scm` 等，切换语言时读入并传给 `JavaLanguageSpec`（`MainActivity.kt:947-960`）。
- Tree-sitter 主题（capture name → TextStyle）：
  - `TsTheme` 支持“最长匹配 scope”降级（如 `variable.builtin.this` 会命中 `variable.builtin`），并把 capture pattern id 映射到 style（`language-treesitter/.../TsTheme.kt:45-87`）。
  - demo 对 Java 的映射写在 `app/src/main/java/.../lang/langJava.kt:221-238`（如把 `"comment"` 映射为 `EditorColorScheme.COMMENT` 且斜体）。

4) **内置示例 Java 高亮（非 TextMate/Monarch/TS）**
- `JavaLanguage`（`language-java/.../JavaLanguage.java:62-158`）使用 `JavaIncrementalAnalyzeManager`。
- `JavaIncrementalAnalyzeManager extends AsyncIncrementalAnalyzeManager`：逐行 tokenizer（`tokenizeLine`），并在 `generateSpansForLine` 把 token 映射为 `EditorColorScheme` 里的类别（`language-java/.../JavaIncrementalAnalyzeManager.java:189-270`、`291+`）。
